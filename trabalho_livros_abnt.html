1. INTRODUÇÃO

No âmbito do desenvolvimento de softwares, as estruturas de dados desempenham um papel fundamental. Elas representam a forma organizacional pela qual os dados são estruturados, armazenados, acessados e manipulados na memória do computador. De acordo com a literatura técnica de computação, a escolha adequada de uma estrutura de dados impacta diretamente na eficiência algorítmica do programa, refletindo no consumo de memória RAM e na velocidade de processamento do processador (CPU).

Dentre as categorias de estruturas de dados existentes, destacam-se as estruturas lineares. Nessas coleções, os elementos estão dispostos de forma sequencial, onde cada item possui um predecessor e um sucessor direto, com exceção do primeiro e do último item da coleção. Os exemplos mais clássicos de estruturas lineares incluem os Vetores (ou Arrays), as Listas Encadeadas, as Filas e as Pilhas. Este trabalho tem como objetivo modelar e implementar uma solução eficiente para o armazenamento e remoção de livros a partir de uma biblioteca virtual baseada em uma lista linear, simulando o processo prático de doação de volumes físicos.

2. DESENVOLVIMENTO

2.1. Modelagem do Problema e Escolha da Estrutura em Ambientes Modernos

O problema apresentado requer o armazenamento de informações relativas a livros (título e autor) acumulados ao longo do tempo. Posteriormente, devido à ação de doar livros, surge a necessidade de remover de forma precisa e eficiente determinados volumes da biblioteca pessoal através do seu identificador único (ID).

A estrutura fundamental por trás dessa solução é a Lista Linear Sequencial (Vetor/Array), um conceito clássico que continua sendo a espinha dorsal do processamento de dados atual. Embora seja uma teoria consolidada, a forma como lidamos com vetores evoluiu significativamente. Em 2026, motores de execução modernos (como a engine V8 do JavaScript e o gerenciador de memória do Python) abstraem a alocação contígua de memória RAM. O desenvolvedor interage com coleções dinâmicas altamente otimizadas, enquanto o compilador ou interpretador lida com a complexidade de alocação física de hardware "por baixo dos panos".

2.2. Operações de Remoção: Do Shifting Clássico à Filtragem Funcional

A operação central deste problema é a remoção de elementos. Na teoria estrutural de dados, quando um elemento em um índice intermediário $i$ é excluído, o array não pode deixar um espaço vazio ("buraco"), pois isso quebraria a integridade da sequência em memória. A evolução da engenharia de software aborda esse problema em dois paradigmas:

  • A Abordagem de Baixo Nível (Shifting Físico): Historicamente, em linguagens como C, o programador precisava escrever laços de repetição manuais para mover fisicamente todos os elementos à direita do item excluído uma posição para a esquerda. Isso ensina como a arquitetura do computador opera a nível de bit e byte.
  • A Abordagem Moderna (Paradigma Declarativo/Funcional): Hoje, em ecossistemas modernos e orientados por IA (Vibe Coding), os programadores raramente lidam com deslocamentos de memória brutos. Utilizam-se funções declarativas (como filter() em JavaScript ou List Comprehensions em Python). O sistema avalia a regra e reconstrói o array limpo, abstraindo a exclusão e aproveitando os abundantes recursos de CPU/RAM de 2026.

2.3. Código-Fonte da Solução

A seguir, são apresentadas as três implementações da solução do problema em diferentes linguagens de programação, cobrindo as duas metodologias explicadas anteriormente.

Implementação em JavaScript (Filtragem Dinâmica)

Código utilizado para a aplicação web interativa, onde a remoção é feita de maneira declarativa utilizando o método filter da API nativa de arrays. 

// Estrutura de dados: Lista de objetos em JavaScript
let biblioteca = [
    { id: 1, titulo: "O Senhor dos Anéis", autor: "J.R.R. Tolkien" },
    { id: 2, titulo: "1984", autor: "George Orwell" },
    { id: 3, titulo: "Dom Casmurro", autor: "Machado de Assis" }
];

// Função de remoção por filtragem lógica (Retorna novo array sem o item)
function removerLivro(id) {
    biblioteca = biblioteca.filter(livro => livro.id !== id);
}

Implementação em Python (Métodos Embutidos)

Em Python, a remoção pode ser feita identificando o índice do elemento e executando o método pop() ou reconstruindo a lista por compreensão (list comprehension). 

# Estrutura de dados: Lista de dicionários em Python
biblioteca = [
    {"id": 1, "titulo": "O Senhor dos Anéis", "autor": "J.R.R. Tolkien"},
    {"id": 2, "titulo": "1984", "autor": "George Orwell"},
    {"id": 3, "titulo": "Dom Casmurro", "autor": "Machado de Assis"}
]

def remover_livro(livro_id):
    global biblioteca
    # Reconstrói a lista sem o elemento de id correspondente
    biblioteca = [livro for livro in biblioteca if livro["id"] != livro_id]

Implementação em C (Vetores e Deslocamento Manual)

A implementação em linguagem C demonstra o algoritmo clássico de deslocamento (shifting) de elementos em um vetor físico de tamanho estático, mantendo a integridade da lista. 

#include <stdio.h>
#include <string.h>

typedef struct {
    int id;
    char titulo[100];
    char autor[100];
} Livro;

Livro biblioteca[100]; // Vetor estático com capacidade de 100 livros
int totalLivros = 3;  // Controle do número atual de elementos

void removerLivro(int id) {
    int indiceEncontrado = -1;
    // Passo 1: Busca sequencial pelo elemento
    for(int i = 0; i < totalLivros; i++) {
        if(biblioteca[i].id == id) {
            indiceEncontrado = i;
            break;
        }
    }
    
    // Passo 2: Deslocamento (shifting) para a esquerda
    if(indiceEncontrado != -1) {
        for(int i = indiceEncontrado; i < totalLivros - 1; i++) {
            biblioteca[i] = biblioteca[i + 1];
        }
        totalLivros--; // Decrementa o número de elementos ativos
    }
}

2.4. Análise de Complexidade de Algoritmos

Analisar a eficiência dos métodos implementados é fundamental para entender o comportamento das estruturas sob grandes volumes de dados. Adotando a notação Big-O, temos as seguintes classificações:

  1. Busca (para exclusão): Para identificar qual elemento deve ser removido pelo seu identificador (ID), o algoritmo realiza uma busca sequencial, iniciando do primeiro elemento até encontrar o correspondente. A complexidade de tempo no pior caso é de $O(n)$, onde $n$ é o total de livros cadastrados.
  2. Remoção com Deslocamento (C): Após localizar o elemento no índice $i$, todos os $n - 1 - i$ elementos subsequentes precisam ser movidos fisicamente. No pior caso (remoção do primeiro livro), a complexidade de movimentação na memória RAM é de $O(n)$.
  3. Remoção por Filtragem (JS e Python): O método reconstrói um novo vetor copiando todos os itens que devem ser mantidos. A complexidade do algoritmo é linear, classificada como $O(n)$ em termos de tempo e $O(n)$ em complexidade espacial, visto que necessita de memória temporária para o novo vetor filtrado.

2.5. O Impacto da Inteligência Artificial e o Paradigma de "Vibe Coding"

O desenvolvimento moderno de softwares está passando por uma revolução impulsionada por assistentes de inteligência artificial generativa. Nesse novo cenário, surge o paradigma do "Vibe Coding" (programação por direcionamento/vibe). Trata-se de um método de desenvolvimento de software onde o programador humano atua principalmente como arquiteto e revisor de sistemas, instruindo modelos de IA a escrever, refatorar e otimizar linhas de código em tempo real.

Sob essa ótica, a inteligência artificial desempenha um papel crítico na otimização de estruturas de dados. Modelos de linguagem de larga escala (LLMs) auxiliam programadores a:

  • Analisar a complexidade assintótica de algoritmos em tempo real (calculando o Big-O automaticamente).
  • Identificar gargalos de processamento, recomendando a transição de vetores lineares ($O(n)$) para tabelas hash ou árvores de busca ($O(\log n)$ ou $O(1)$) quando o volume de dados exige maior eficiência.
  • Refatorar algoritmos de remoção manuais (como deslocamentos em C) para soluções idiomáticas mais limpas em linguagens modernas (como filtragens declarativas em Python e JavaScript).

2.6. Estudo de Caso Prático: Estruturas de Dados no Sistema Nexus AI (Dublagem)

Para contextualizar a teoria na prática de engenharia de software de ponta, o autor deste trabalho utiliza e desenvolve o sistema **Nexus AI**, uma aplicação em ambiente Python destinada à dublagem automatizada de vídeos e jogos utilizando modelos locais de IA (como Gemma 4 e Qwen 3 TTS) acelerados por hardware (RTX 3050). A arquitetura deste software depende diretamente de diversas estruturas de dados integradas no arquivo nexus_core.py para viabilizar processamento multimídia eficiente:

  1. Tabelas Hash / Dicionários (dict): Utilizados para mapear os perfis de jogos (GAME_PROFILES), associando IDs de jogos (chaves) a configurações dinâmicas de áudio e instruções de IA (valores). Essa estrutura permite acesso rápido em tempo de busca constante $O(1)$, essencial para evitar delays durante o processamento em tempo de execução. Também é aplicada no glossário fonético de sotaques e correções textuais.
  2. Dicionários Ordenados (OrderedDict): Empregados na organização de embeddings de voz e segmentos de fala gerados na diarização do áudio (reconhecimento de quem está falando). A ordenação de inserção é fundamental para garantir a sincronia e a linearidade cronológica da dublagem.
  3. Vetores Dinâmicos e Matrizes (list e np.ndarray): Utilizados para armazenar caminhos de arquivos temporários de áudio, listas de dependências e representações matemáticas (tensores e matrizes tridimensionais) de ondas sonoras carregadas na memória RAM pelo PyTorch e Numpy. As operações de corte de silêncio e filtragem de segmentos de áudio operam em tempo linear $O(n)$, semelhante ao algoritmo de remoção do acervo de livros.
  4. Estruturas de Controle de Concorrência (threading.Lock): O Nexus AI utiliza múltiplas threads em paralelo para aceleração local. Para impedir que duas threads tentem acessar os motores de IA simultaneamente (causando colisão de memória na VRAM da GPU), são utilizados mecanismos de sincronização chamados Locks (ex: gema_lock e o lock de controle de concorrência do motor de voz TTS), que enfileiram as requisições de forma estruturada.

3. CONCLUSÃO

O desenvolvimento deste projeto permitiu consolidar na prática os fundamentos teóricos de estruturas de dados lineares e suas operações de manipulação em memória. A modelagem do armazenamento da biblioteca pessoal e o procedimento de doação demonstraram que a remoção de itens em uma lista linear baseada em vetores exige atenção direta à integridade dos índices.

Conclui-se que para acervos de tamanho reduzido ou moderado (como uma biblioteca pessoal), a utilização de Arrays e Listas sequenciais nativas de linguagens de alto nível (com complexidade de remoção $O(n)$) mostra-se extremamente simples e eficiente devido à otimização a nível de compilador. No entanto, para sistemas de larga escala que lidam com milhões de registros e frequentes operações de exclusão, estruturas alternativas como as Listas Encadeadas (onde a exclusão física é feita alterando ponteiros sem necessidade de deslocamento de memória) ou as Tabelas Hash devem ser consideradas para otimizar os recursos do computador.
